# 系统健壮性与体验迭代（判分统一 / 错误提示 / hash 路由等）

日期：2026-07-03

本轮不加新功能，聚焦系统设计与日常使用中的 6 个痛点。

## 1. 判分逻辑统一到 lib/grade.js，并做宽容归一化

**问题**：`isCorrect` 在 server.js、routes/srs.js、routes/exam.js 各复制了一份，判分标准可能随改动漂移；且填空题判分只做 trim+小写，以下常见输入会被误判为错：

- 手机键盘自动替换的弯撇号：`isn’t`（正确答案是 `isn't`）
- 句尾多写/少写的 `.` `?` `!`
- 词间多敲的空格、全角空格

**改动**：新增 `lib/grade.js` 导出 `normalizeAnswer` / `isCorrect`，三处调用方全部改为引用它。归一化规则：弯引号→直引号、全角空格→半角、去句尾标点、压缩连续空格、忽略大小写。routes/transform.js 保留自己更宽的专用归一化（n't/not 缩写等价），不受影响。

## 2. 前端 api() 统一错误处理

**问题**：`api()` 原来是 `fetch().then(r => r.json())`，服务未启动、接口 500/404 时前端静默失败，界面停在「加载中…」没有任何提示。

**改动**：`public/app.js` 的 `api()` 检查网络错误与 `r.ok`，失败时 toast 出服务端 `error` 字段或状态码；同一错误 3 秒内只弹一次（避免首页并发请求失败时连弹）。所有 feat 模块经 `NCE.api` 复用，自动受益；原有 `.catch(() => fallback)` 兜底逻辑不受影响。

## 3. /api/questions 支持 countOnly=1

**问题**：出题设置页每点一次筛选 chip 就全量下载当前条件下的所有题目（含题干/选项），只为了显示「共有 N 道题」。

**改动**：`countOnly=1` 时只返回 `{count}`；`updatePoolHint` 改用该参数。

## 4. Hash 路由：刷新不丢标签页

**问题**：刷新页面永远回到首页；也无法把「背单词」「听写」等页面存成书签/链接发出去。

**改动**：点击任意标签/子页时把 id 写入 `location.hash`（`replaceState`，不污染历史记录）；载入与 `hashchange` 时按 hash 恢复标签页（子页如 `#vocab`、`#words` 也支持，经宿主标签自动展开）。答题草稿恢复时 hash 同步为 `#practice`，保持地址栏与页面一致。

## 5. 「练本课语法」优先出本课的题

**问题**：原实现按语法标签在全册取题，练第 3 课可能抽到第 121 课的题，与当前学习进度脱节。

**改动**：先取挂在本课的题（新概念1偶数课是配套练习，挂在 n+1 课号的题也算本课），不足 10 题再用本课语法标签的同册题随机补足。

## 6. 结果页告知错题去向

**问题**：交卷后错题被静默加入间隔复习队列，用户不知道该去哪复习。

**改动**：成绩单副标题追加「N 道错题已加入『🔁 间隔复习』，到期后按遗忘曲线重现」。
