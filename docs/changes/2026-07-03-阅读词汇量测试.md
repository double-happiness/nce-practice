# 新功能：阅读词汇量测试 + 听读词汇量增强

日期：2026-07-03

## 是什么

在已有「👂 听力词汇量」旁新增独立的「📖 阅读词汇量」模块：显示英文单词，4 选 1 选中文释义，用同一套分层抽样 + 猜中基线剔除算法估算「看得懂」的词汇量。

同时补齐：动态册别选择、按册筛选历史、首页词汇量基线卡片、备份概览。

## 阅读 vs 听力

| | 听力词汇量 | 阅读词汇量 |
| --- | --- | --- |
| 刺激 | 只放发音，不显示单词 | 显示英文单词 + 词性 |
| 估算对象 | 听得懂 | 看得懂 |
| 数据文件 | `listen-vocab.json` | `read-vocab.json` |

## 实现

| 文件 | 内容 |
| --- | --- |
| `lib/vocabtest.js`（新增） | 听/读共用：分层抽样、选项、估算、`/books` 可用册列表、`/history?book=` 按册筛选 |
| `routes/listenvocab.js` | 改为薄封装，行为不变 |
| `routes/readvocab.js`（新增） | 阅读词汇量 API |
| `public/js/feat-readvocab.js`（新增） | 阅读词汇量前端三态 UI |
| `public/js/vocabtest-ui.js`（新增） | 册别下拉、历史展示共用 |
| `public/app.js` | 首页「词汇量基线」卡片；单词合并标签纳入 `readvocab` |
| `routes/backup.js` | 备份 KEYS 加入 `read-vocab` |
| `public/js/feat-backup.js` | 概览显示听/读测次 |

## 验证

- `npm run validate` 0 错误
- `/api/read-vocab/*` 与 `/api/listen-vocab/*` 端到端可用
- `/api/*/books` 返回第 1 册（360 词）、第 2 册（618 词）

## 后续增强（同批次）

- `GET /api/vocab-test/overview?book=` 合并听/读估算、近 3 次平均、听读差距
- 首页词汇量卡片深链到对应册别；结果页与入口页显示听读对比
- 背单词册别下拉与词汇量测试共用动态册列表
