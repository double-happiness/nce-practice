# Change：题库文本批量导入工具

**日期**：2026-07-03

## 需求

扩题一直靠手写 JSON（引号、逗号、id 手动编号），容易出错且效率低。希望有工具化的录入方式。

## 方案

`lib/data.js` 已支持 `data/questions/` 分片自动合并，因此只需解决「不手写 JSON 也能产出合法分片」。新增 `scripts/import.js`：用紧凑纯文本写题，脚本负责转 JSON、自动编号、校验后落盘。

## 草稿格式

```
# book=1 lesson=7 grammar=冠词a/an
He is ___ engineer.
* an | a | the | -
解析: engineer 以元音音素 /e/ 开头，不定冠词用 an。

She ___ (drink) tea every morning.
答案: drinks
解析: 第三人称单数一般现在时，drink 加 -s。
```

- `#` 行设置公共字段（book / lesson / grammar），对后续题目生效，可中途覆盖；空行分隔题目
- 选项行用 `|` 分隔、正确答案前加 `*`；无选项行即为 fill 填空题，用「答案:」给答案（多个可接受答案用 `|` 分隔，输出为数组）
- 「语法:」行可按题覆盖公共 grammar；「解析:」行为讲解

## 用法

```
npm run import -- 草稿.txt [--out 批次名] [--dry-run] [--force]
```

- 输出到 `data/questions/<批次名>.json`（默认取草稿文件名），已存在则拒绝
- id 沿用主流格式 `b{册}-{课}-q{N}` / `b{册}-{课}-f{N}`，扫描现有全部题目按 册+课+题型 自动续号
- `lessonTitle` 从课程数据按 book+lesson 自动带出，查不到则警告并留空
- 题干与现有题库重复默认报错（`--force` 降级为警告），防止重复录入
- 写盘后跑全量 `validate()`，与导入前基线相比出现新错误则自动删除文件回滚

## 其它改动

- `scripts/validate.js`：新增题干查重警告（忽略大小写与多余空白）——上线即发现存量重复：`b2-10-q3` 与 `b2-005` 题干相同。已删除 `b2-005`（挂在占位课名"被动语态"下、解析较简），保留信息更准的 `b2-10-q3`（真实课程 L10 "Not for jazz"）；两个 id 均未被用户进度数据引用，删除无副作用。另用更宽松的近似匹配（忽略标点）全量扫过一遍，全库仅此一组重复
- `package.json`：新增 `import` script

## 验证

- dry-run：3 题（2 mcq + 1 fill）正确解析，id 续号为 `b1-7-q3` / `b1-9-q6` / `b1-9-f1`，lessonTitle 自动带出，多答案填空输出数组
- 真实导入后 `npm run validate` 0 错误；重名分片、重复题干、缺 `*` 答案、缺 book/lesson 均按预期报错并带行号
- 测试分片已删除，题库恢复原状
