# 新功能：听力词汇量测试

日期：2026-07-03

## 是什么

「单词」标签下新增「👂 听力词汇量」子页：**只放发音、不显示单词**，从 4 个中文释义中选出词义，测完估算你在本册词表中「听得懂」的词汇量（区别于「背单词」测的是看得懂/拼得出）。

## 测试与估算方法

1. **分层抽样**：全册去重词表（第 1 册 360 词，与背单词共用同一口径）按课程先后切成 3 段难度带（新概念越往后词越难），每段随机抽同样多的词。题量可选 12 / 24 / 36。
2. **答题**：自动播放发音（可重听），从 4 个中文释义里选；干扰项优先取同词性的其他词释义，且互不重复。有「听不出来，跳过（算错）」的诚实选项。答完立即揭晓单词、音标、释义、例句。
3. **估算**：每段正确率先剔除 4 选 1 的猜中基线——`adj = max(0, (p − 25%) / 75%)`，再乘以该段词表总数，三段求和。全靠瞎蒙的估算值≈0，不会虚高。
4. **结果页**：估算词汇量大数字 + 各难度段进度条（答对数 → 折算词数）+ 没听出来的词清单（带发音、可**一键全部加入生词本**，汇入现有复习闭环）。
5. **历次成绩**：按档案存 `listen-vocab.json`（上限 50 条），入口页显示上次估算与近几次记录；已纳入「数据备份」导出/导入与每日快照。

## 实现

| 文件 | 内容 |
| --- | --- |
| `lib/dict.js`（新增） | 去重词表 `buildDict(book)` / `normKey`，从 routes/words.js 抽出共用，保证「词汇总量」口径一致 |
| `routes/words.js` | 改用 lib/dict.js（行为不变） |
| `routes/listenvocab.js`（新增） | `GET /listen-vocab/sample`（分层抽样+选项生成）、`POST /listen-vocab/grade`（判分，答案在服务端）、`POST /listen-vocab/finish`(汇总估算+写历史)、`GET /listen-vocab/history` |
| `public/js/feat-listenvocab.js`（新增） | 前端模块：入口页/答题页/结果页三态，键盘 1-4 选项、回车下一词，状态保持（切标签回来不丢进度） |
| `public/app.js` | `listenvocab` 并入「单词」合并标签；首页快捷入口加「👂 听力词汇量」 |
| `public/index.html` | 引入 feat-listenvocab.js |
| `routes/backup.js` | 备份 KEYS 与概览加入 `listen-vocab` |

## 验证（端到端实测）

- 抽样：24 题，3 段各 120 词（L1–41 / L43–89 / L89–143），全部题目 4 个选项且不重复
- 判分：选对 / 选错 / 跳过三条路径正确；不存在的词返回 404 JSON
- 估算：模拟「前段全对、中段一半对、后段全错」→ 120 + 60 + 0 = 180/360，与公式一致
- 历史：写入并可查询；测试产生的假记录已清理
- `npm run validate` 0 错误 0 警告

## 已知边界

- 发音用浏览器 TTS（与听写一致），音色因浏览器而异；语速跟随顶栏全局语速设置。
- 估算精度随题量提高，同一题量多测几次取平均更稳（结果页已注明）。
- 目前仅第 1 册（词表口径 360 词）；2–4 册课程收录后自动可用，无需改代码。
