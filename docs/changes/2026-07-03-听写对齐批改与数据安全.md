# 听写对齐批改 + 服务端硬化 + 每日数据快照

日期：2026-07-03（当日第二轮迭代，接「系统健壮性与体验迭代」）

## 1. 听写批改升级（public/js/feat-dictation.js）

**问题一（bug）**：`tokenize` 只保留直撇号 `'`，手机/Mac 键盘自动替换出的弯撇号 `’` 会被当作分隔符——用户打 `don’t` 被拆成 `don t` 两个词，与标准答案 `don't` 错位，判错。

**问题二（体验）**：原来是逐位比对，漏听或多打一个词后，后面所有词全部错位标红，一句 10 个词漏 1 个只得 0–20%，打击感极强且不反映真实水平。

**改动**：

- `tokenize` 先把 `‘ ’ ʼ` 归一为 `'`；
- `compare` 改为**最长公共子序列（LCS）对齐**后批改：漏词/多词/错词只影响那一处，其余照常判对。
- 纯函数移到 NCE 守卫之前并可被 Node `require`，补了 6 个单测用例（弯撇号、漏词、多词、错词、空作答、大小写标点），全部通过。

## 2. 服务端 API 硬化（server.js）

- **JSON 错误兜底**：Express 默认对路由异常返回 HTML 错误页、对未知路径返回 HTML 404，前端 `api()` 无法解析。现在未知 `/api/*` 返回 `{error:"接口不存在：…"}`，未捕获异常返回 `{error:"服务器内部错误…"}` 并打服务端日志，与上一轮的前端错误 toast 打通。
- **请求体上限 100kb → 10MB**：备份导入（`POST /api/backup/import`）上传整份学习数据，记录多了以后必超默认限制导致导入失败；超限时返回明确的 413 JSON。

## 3. 学习数据每日滚动快照（lib/snapshot.js）

**问题**：全部学习记录存在 `data/profiles/` 的 JSON 文件里，原子写只防进程崩溃，防不了误操作——手滑「重置进度」或导入一份坏备份，数据就没了，唯一保护是用户记得手动导出。

**改动**：服务启动时（及每 6 小时定时器兜底）把 `data/profiles/` 整体复制为 `data/backups/YYYY-MM-DD/`，先复制到 `.tmp` 再改名保证快照完整性；每天最多一份、滚动保留最近 7 份；`.trash`、`*.tmp.*` 不入快照。目录已加入 `.gitignore`。

**恢复方式**：停服后把某天的目录内容拷回 `data/profiles/` 即可，例如：

```bash
cp -R data/backups/2026-07-03/ data/profiles/
```
