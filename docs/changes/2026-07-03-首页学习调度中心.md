# 首页升级为学习调度中心（薄弱专练直达 + 单词复习汇入今日任务）

日期：2026-07-03（当日第三轮迭代）

## 目标

薄弱分析、单词掌握度这些数据早就有了，但都躺在各自的标签页里，离每天的学习动线太远。本轮把首页「今日学习」升级成调度中心：**每天打开就知道该干什么、一键直达**。

## 改动

### 1. 新增「📌 薄弱专练」卡片（public/app.js renderHome）

- 从 `/api/stats/grammar` 取语法正确率统计，筛选「练够 4 次且正确率 < 85%」的最弱语法点（练 1 次错 1 次不算薄弱，避免误导）。
- 卡片显示：最弱语法点 + 正确率（红黄绿配色）+ 练习次数，若有第二薄弱点一并提示。
- 「专练 10 题 →」按钮调用新增的 `practiceGrammar(tag)`：按该语法点随机组 10 题直入答题，并把刷题设置页的筛选（册数/语法点）同步过去——练完点「再练一组」自动延续同一条件。
- 无薄弱点时显示鼓励文案 + 薄弱分析入口；无练习数据时引导先刷题。

### 2. 「🔁 到期复习」卡片升级为「今日复习」

- 原来只有 SRS 到期错题数；现在合并显示 `/api/words/stats` 中「学习中」（level 1–2）的待巩固单词数。
- 新增「默写单词 →」按钮，通过 `NCE.pendingWords = { mode: 'spell' }` 深链直接落到「背单词」的默写模式（feat-words.js 的 onShow 消费该标记，机制与听写深链 `NCE.pendingDictation` 一致）。

### 3. 首页健壮性

- `renderHome` 的 5 个数据请求逐项 `.catch` 兜底：任一接口失败只影响对应卡片的数据，不再让整个首页卡在「加载中…」。

### 4. 顺手修的一致性问题

- `beginQuiz` 现在同步 `location.hash` 为 `#practice`：从首页「薄弱专练」、教材页「练本课语法」直入答题时，刷新页面不再跳错标签。

## 涉及文件

- `public/app.js`（renderHome 重构 / practiceGrammar / beginQuiz hash 同步）
- `public/js/feat-words.js`（pendingWords 深链消费）
- `public/styles.css`（`.hc-btns` 双按钮排布）
- `README.md`

## 验证

- `/api/stats/grammar` 返回 `{tag, seen, accuracy}`，薄弱筛选逻辑基于 seen≥4 且 accuracy<85
- `/api/questions?grammar=be动词&book=1&countOnly=1` → 32 题，组题链路可用
- `/api/words/stats?book=1` → `{learning}` 待巩固池可用
- `node -c` 全部通过
