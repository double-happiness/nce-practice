# Change：句型转换训练

日期：2026-07-03
范围：新功能模块（数据 + 后端路由 + 前端标签页）

## 背景

第一轮遗留建议中的「题型扩展（句型转换/翻译）」。训练流程：给出中文 → 译成英文 → 依次改一般疑问句、否定句、对句子不同成分提问（特殊疑问句）。

## 数据

- 新增 `data/transforms/` 分片目录（`lib/data.js` 自动合并，同题库/课程的分片机制）。每条练习：
  ```jsonc
  {
    "id": "tf-b1-001", "book": 1, "lesson": 1, "grammar": ["be动词"],
    "cn": "这是一本书。",
    "explanation": "转换规则讲解（末步展示）",
    "steps": [
      { "kind": "translate|yesno|negative|wh", "prompt": "步骤要求", "answers": ["可接受答案", "…"] }
    ]
  }
  ```
- 种子内容 20 句 94 步：第 1 册 14 句（be 动词 / there be / 现在进行时 / 一般现在时 / can / 一般过去时 / be going to / how many / 颜色 / 频率等），第 2 册 6 句（过去时 / 现在完成时 / 过去进行时 / will / 被动语态），课次锚点与既有 grammar 标签对齐。
- `scripts/validate.js` 新增 transforms 校验：id 唯一、kind 枚举、prompt/answers 非空；首步非 translate 给警告。

## 后端（routes/transform.js，自动挂载）

- `GET /api/transform/meta` — 各册练习数。
- `GET /api/transform/exercises?book=&lessonMin=&lessonMax=&limit=&random=1` — 出题，**不含 answers/explanation**。
- `POST /api/transform/grade` `{id, step, response}` — 单步判分，返回 `{correct, answers, isLast, explanation}` 并记录进度。
- `GET /api/transform/stats` — 当前档案累计步数/正确率/按步骤类型拆解。
- 判分归一化：大小写、标点（撇号除外）、多余空格不敏感；`n't` 缩写与 `not` 全形等价（can't/won't/cannot 特判）；弯引号转直引号。`isn't/What's` 等 `'s`/be 缩写变体由题目数据显式列出，避免 `'s`＝is/has 歧义误判。
- 进度按档案存 `data/profiles/<id>/transforms.json`；`routes/backup.js` 的备份键新增 `transforms`。

## 前端（public/js/feat-transform.js）

- registerFeature 自注册「🔀 句型转换」，归入「练」分组（app.js `TAB_GROUP_OF` 增加一行）。
- 流程：选册/句数/随机 → 每句按步骤作答（回车提交、回车继续），判分后显示 ✓/✗、参考答案（可 🔊 朗读）、末步显示转换规则讲解；已完成步骤以「转换链」列表展示标准形式，转换过程一目了然。
- 小结页：本轮正确率 + 按步骤类型（中译英/一般疑问句/否定句/特殊疑问句）拆解。
- 备份页数据概览新增「句型转换步数」。

## 验证

- `npm run validate` 0 错误 0 警告；全部改动文件 `node --check` 通过。
- API 级 E2E：出题无答案泄露；宽容判分（小写无句号、isn't/can't/cannot、弯引号 What's）全部判对；错误答案判错；step 越界/未知 id 返回 4xx；stats 与 backup/info 正确累计。
- 无头 Chrome E2E（puppeteer-core，装于临时目录未入项目依赖）：标签渲染于「练」分组、册数 chips、开始训练、正确/错误提交反馈、步骤推进与转换链展示，无 JS 错误。
- 测试写入的 `data/profiles/default/transforms.json` 已删除，不影响真实学习记录。

## 注意

# 迭代 2：SRS 集成 + 每日目标计入 + 高级转换扩充（同日）

上轮遗留的三个迭代方向全部落地：

## 1. 错步进入间隔复习队列

- 新增 `lib/transform-util.js`：判分归一化与 kind 定义收敛为单一来源（transform 路由、SRS 路由、validate 共用）；提供 `resolveStep`（解析 `tf-xxx#步骤下标` 形式的 key）与 `stepQuestion`（构造复习用的自含伪题——非首步 stem 带上标准原句，脱离转换链也能作答）。
- `routes/srs.js`：`/srs/add`、`/srs/due`、`/srs/grade` 全部兼容转换步骤 key；判分走转换归一化，反馈带参考答案与转换规则讲解。题库题行为不变。
- `feat-transform.js`：答错时前端自动 `POST /srs/add`（与刷题/测验错题入队走同一约定），反馈提示"已加入间隔复习"。
- `feat-review.js`：复习界面对 `type: 'transform'` 走已有的自由输入分支，**零改动**即可作答；仅更新了说明文案。

## 2. 训练量计入「今日学习」每日目标

- `routes/plan.js` `/plan/overview`：转换训练的步骤记录（`{ts, correct}` 与刷题 attempts 结构兼容）并入聚合，打卡/连续天数/30 天日历/今日进度条一并生效，前端零改动。

## 3. 内容扩充（20 → 31 句，新增 kind：passive / indirect）

- `data/transforms/b1-batch2.json`（4 句）：some/any + How much、比较级选择疑问、What time 时刻提问、will 将来时。
- `data/transforms/b2-batch2.json`（7 句）：过去完成时、第一条件句（if 从句不动只动主句）、must（否定语义特殊故不练否定，讲解说明 needn't/mustn't 区别）、主动转被动 ×2（was built / is spoken）、间接引语 ×2（时态后移、will→would + tomorrow→the next day）。
- kind 枚举扩展为 translate/yesno/negative/wh/indirect/passive，validate 从 `lib/transform-util` 取单一来源。

## 验证（迭代 2）

- `npm run validate` 0 错误 0 警告；全部改动文件 `node --check` 通过。
- API 级 E2E：错步判错→入队→due 返回自含伪题→SRS 宽容判分通过→非法 key（不存在的练习/越界步骤）拒绝→题库题入队不受影响→`/plan/overview` todayCount 正确计入转换步数→passive/indirect 新 kind 判分正常。
- 无头 Chrome 冒烟：原有流程无回归，前端答错自动入队（stats 队列数随之 +1），无 JS 错误。
- 测试写入的 `data/profiles/default/transforms.json`、`srs.json` 已清理。
