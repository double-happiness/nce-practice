# Change：学习状态服务器化与档案隔离收尾

日期：2026-07-03
范围：多用户档案功能收尾（前端存储层 + 后端新路由 + 备份 + git 数据卫生）

## 背景

多用户档案功能上线后,后端学习数据已按 `data/profiles/<档案id>/` 隔离,但前端 localStorage 里的轻量学习状态(最近学习课、已学 ✓、已掌握 ★、听写成绩与断点)仍是设备级共享,切换档案会串数据。分两步收尾:先按档案隔离,再服务器化。

## 第一步:cookie 下发档案 id + localStorage 按档案加前缀

- `server.js` 新增中间件,每个响应 `Set-Cookie: nce_pid=<当前档案id>`,前端可**同步**读取,避免异步 fetch 竞态。
- 新增 `public/js/profile-store.js`(在所有业务脚本之前加载),`default` 档案不加前缀(旧数据零迁移),其它档案 key 形如 `<pid>:<key>`。
- 设备偏好(`nce-theme` 主题、`nce-tts-rate` 语速)保持全局共享,不隔离。

## 第二步:服务器化(NCEStore)

- 新增 `routes/uistate.js`:
  - `GET /api/ui-state` — 当前档案全部 UI 学习状态;
  - `POST /api/ui-state` — 浅合并部分更新,白名单 6 键(`nce-last-lesson` / `nce-viewed-lessons` / `nce-mastered-lessons` / `dct-best` / `dct-last` / `dct-pos`),非法键/非对象体返回 400。
  - 数据存 `data/profiles/<id>/ui.json`(`writeJSONAtomic` 原子写),`lib/profile.js` 的 `USER_FILES` 纳入 `ui.json`。
- `profile-store.js` 升级为完整存储 `NCEStore`:
  - 启动先用 localStorage 镜像同步填充内存缓存,再拉取 `/api/ui-state`;
  - **服务器完全为空** → 首次启用,本地旧数据一次性全量迁移上传;
  - **服务器非空** → 一律以服务器为准,镜像同步成服务器状态(含删除服务器没有的键,避免恢复备份后被本地旧镜像"复活"已删数据);
  - `set()` 写穿透:内存 + 本地镜像 + POST 服务器;服务器不可达时降级用镜像,数据不丢。
- `app.js` / `feat-dictation.js` 相关读写全部改走 `NCEStore.get/set`;`init()` 首行 `await NCEStore.ready`,首屏渲染前状态就绪。`dct-best` 从字符串改存数字(读取处 `parseFloat` 兜底旧数据)。

## 备份

- `routes/backup.js` 的导出/导入/概览纳入 `ui`。
- `feat-backup.js` 导入成功后 1.5s 自动 `location.reload()`(此前只提示"建议刷新",NCEStore 内存缓存会停留在旧值)。

## Git 数据卫生

- 初始 commit 误跟踪了个人学习数据(`data/profiles/default/progress.json`、`vocab.json`、`profiles.json`),已 `git rm --cached` 移出索引(本地文件保留)。
- `.gitignore` 从档案迁移前的旧路径改为整体忽略 `data/profiles/`;`data/transforms/` 是题库内容,保持入库。

## 取舍与已知边界

- 并发写是"整键覆盖、后写赢",单人本地应用足够;多端同时在线编辑需再升级为服务端合并。
- "服务器非空时本地独有键会被删除"意味着:若在第二个浏览器上有第一个浏览器迁移时没有的旧 localStorage 数据,不会二次合并上传。单人单浏览器场景无影响。

## 验证

- `/api/ui-state` CRUD、白名单校验、按档案隔离、备份导出含 `ui` 均经 curl 实测。
- `profile-store.js` 合并/迁移/写穿透逻辑经 Node 桩环境断言(空库迁移、服务器为准含删除、set 写穿透)。
