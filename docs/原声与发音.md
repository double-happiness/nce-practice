## 一、怎么用（学习者）

### 1.1 三种听音方式

系统里所有 **🔊 小喇叭** 共用同一套播放逻辑，按优先级自动选择：

| 优先级 | 来源 | 何时使用 |
|--------|------|----------|
| 1 | **新概念英音原声** | 当前课文有对应 LRC 句子，且你点的内容与原文一致 |
| 2 | **真人单词发音** | 单个词或短短语（Wikimedia 库，约 1000+ 词） |
| 3 | **浏览器 TTS** | 以上都没有时，用系统内置语音朗读 |

**举例**

- 在 **第 1 册 Lesson 1** 点语法例句 *Excuse me!* 的 🔊 → 播放录音里真人朗读者说的原句。  
- 点单词 *handbag* 的 🔊 → 播放单词真人发音（原声课文里是整句，没有单独 handbag 片段）。  
- 在 **听写** 里，若句子与 LRC 原文一致，🔊 同样优先原声。

顶栏 **🔊 语速** 下拉只影响 **TTS**；原声与真人发音按录音原速播放。

### 1.2 教材页 · 🎧 原声课文

打开 **📖 教材学习** → 进入某一课 → 课文区有三个子标签：

| 标签 | 说明 |
|------|------|
| **🎧 原声课文** | 新概念 **英音** 整课 MP3 + LRC 字幕；可整段播放，**点击某句跳转**到对应位置并高亮 |
| **📖 教材原文** | 自行粘贴的课文（可选） |
| **✍️ 原创精读** | 系统生成的同主题精读短文 |

有原声的课会自动显示 **🎧 原声课文** 并默认打开。

在 **原创精读 / 教材原文** 页：

- 每句旁的 🔊：能匹配 LRC 时播原声片段，否则 TTS。  
- **🔊 朗读全文**：本课有原声时直接播整课 MP3；否则 TTS 逐句连读。

### 1.3 覆盖范围

| 册 | 原声 MP3 | 系统收录课次 | 说明 |
|----|----------|--------------|------|
| 第 1 册 | 72 课 | 72（L1–L143 奇数课） | 全覆盖 |
| 第 2 册 | 96 课 | 96 | 全覆盖 |
| 第 3 册 | 60 课 | 60 | 全覆盖 |
| 第 4 册 | 48 课 | 24（L1–L24） | 系统有的 24 课均有原声 |

共 **276** 个 MP3，随仓库分发，**克隆后无需再导入**。

### 1.4 离线

- 原声 MP3、LRC 索引、真人单词库均在 `public/audio/` 下，**不依赖外网**。  
- 安装 PWA 或用过一次后，Service Worker 会缓存索引 JSON；MP3 在播放时由浏览器缓存。  
- 断网时：原声 / 真人发音正常；TTS 需本机已安装英文语音包。

## 二、仓库与离线（维护者 / 新机器克隆）

### 2.1 克隆时必须启用 Git LFS

原声 MP3（约 **550MB**）由 **Git LFS** 托管，普通 `git clone` 只会得到指针文件，无法播放。

```bash
# 1. 安装 Git LFS（只需一次）
# macOS:  brew install git-lfs
# Windows: 见 https://git-lfs.com

git lfs install

# 2. 克隆
git clone git@github.com:double-happiness/nce-practice.git
cd nce-practice

# 3. 若已克隆但未拉 LFS，补拉：
git lfs pull

# 4. 启动
npm install
npm start
```

验证：确认 `public/audio/nce/b1-l001.mp3` 大小约 **600KB+**（不是几百字节的文本指针）。

### 2.2 相关文件一览

```
public/audio/
├── nce/                      # 276 课英音 MP3（Git LFS）
│   └── b{册}-l{课}.mp3       # 例：b1-l001.mp3、b2-l042.mp3
├── official-segments.json    # 每句 LRC 时间轴（🔊 片段匹配用）
└── pronunc/                  # Wikimedia 真人单词发音
    └── *.mp3
data/official/
└── passages.json             # 276 课 LRC 全文 + 音频路径
```

### 2.3 API（可选）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/official/meta` | 已导入原声统计（按册） |
| GET | `/api/official/:book/:lesson` | 单课：title、audio、lines[{t, en}]、en |

## 三、更新原声（维护者）

仅在 **更换/更新源录音** 时需要，日常学习者不用运行。

### 3.1 从本地源目录重新导入

源目录结构需与《新概念英语》英音包一致（含 LRC），默认路径：

`../New_Concept_English/英音/`

```bash
# 复制 MP3 到 public/audio/nce/，并刷新 passages.json、official-segments.json
npm run import:official -- --source "/path/to/英音"

# 强制覆盖已有 MP3
npm run import:official -- --source "/path/to/英音" --force

# 仅开发机省空间：符号链接（不入库，勿用于发布）
npm run import:official -- --source "/path/to/英音" --link
```

### 3.2 提交到 Git

```bash
git add data/official/passages.json public/audio/official-segments.json public/audio/nce/
git commit -m "chore(official): 更新原声课文"
git push    # LFS 会自动上传 MP3
```

`.gitattributes` 已配置 `public/audio/nce/*.mp3` 走 LFS。

## 四、常见问题

**Q：🔊 点了还是机器声？**  
A：可能原因：① 内容与 LRC 原文不完全一致（标点、措辞不同）；② 单词/短语无对应整句原声，已降级到 TTS；③ 克隆时未 `git lfs pull`，MP3 缺失。

**Q：🎧 原声课文标签不显示？**  
A：该课尚未收录原声，或 LFS 文件未拉取成功。检查 `public/audio/nce/` 下是否有对应 `b*-l*.mp3`。

**Q：播放失败 toast？**  
A：确认 MP3 为真实音频文件（非 LFS 指针），路径 `public/audio/nce/b{册}-l{课}.mp3` 存在。

**Q：听写为什么有时不是原声？**  
A：听写句来自精读例句、单词例句等，若与 LRC 课文句不一致，则无法匹配原声，会用 TTS。

**Q：和 `fetch:pronunc` 的关系？**  
A：`npm run fetch:pronunc` 抓取 Wikimedia **单词**发音；`import:official` 处理 **课文整段**原声。两者互补，🔊 自动择优。
