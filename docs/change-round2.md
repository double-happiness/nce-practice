## 2. 多用户档案系统

**背景**：此前所有学习数据全局一份，多人共用同一服务会互相污染进度/错题/打卡。

### 存储结构
- 新增 `lib/profile.js`：档案元信息存 `data/profiles/profiles.json`；六类**按档案隔离**的数据文件（progress / srs / plan / exams / vocab / words）存 `data/profiles/<档案id>/`。
- **自动迁移**：服务首次启动时把 `data/` 根目录下的旧数据文件原子移动到 `data/profiles/default/`，旧用户无感升级。
- 删除档案不直接删数据，移入 `data/profiles/.trash/<id>-<时间戳>/` 防误删。
- 题库、教材内容、meta 仍为全局共享。

### 改动文件
- `lib/progress.js`、`routes/srs.js`、`routes/plan.js`、`routes/exam.js`、`routes/vocab.js`、`routes/words.js`：文件路径从静态常量改为每次请求解析 `profile.file(...)`。
- `routes/backup.js`：导出/导入/概览作用于**当前档案**；备份键新增 `exams`（此前阶段测验历史不在备份内，属遗漏）。
- 新增 `routes/profile.js`（自动挂载）：`GET /api/profile/list`、`POST /api/profile/switch | create | rename | remove`（不能删当前档案、至少保留一个）。
- 新增 `public/profile.js`：顶栏「👤 档案」下拉（独立自注册模块，零依赖 app.js），支持切换（✓ 标当前）、新建（自动分配头像 emoji 并切换过去）、重命名、删除（confirm 确认）；操作后整页刷新以载入新档案数据。

### 已知边界
- 前端 localStorage（最近学习课、已学标记、听写最好成绩、主题、语速）目前仍为**设备级共享**，未按档案隔离；主题/语速本就该共享，学习类标记后续可按档案前缀迁移。
- 正在运行的旧服务进程需**重启**才会走新存储路径；重启前旧进程仍会写 `data/` 根目录（会被下次启动的迁移忽略——迁移只在 profiles.json 不存在时执行一次），期间产生的少量记录可手工合并。
