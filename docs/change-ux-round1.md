# Change：用户体验四项改进（第一轮）

日期：2026-07-03
范围：题库数据、首页信息架构、答题流程、交互细节
背景：从使用者视角评审系统后确定的四项改进，依次实施。

## 1. 补齐第 1 册题库缺口 + 界面如实标注支持范围

**问题**：第 1 册前 25 课有 7 个课对（L3/5/9/11/15/17/19）完全没有题目，另有 5 课题量 1–3 道；而页脚宣传"覆盖 1–4 册"，与实际（第 2 册 20 题、第 3 册 10 题、第 4 册 6 题）不符。

**改动**：
- 新增 `data/questions/b1-001-025.json`（51 题，mcq+fill），第 1 册题量 346 → 397，L1–25 每个课对都有 ≥4 题；`npm run validate` 通过。
- `/api/meta` 新增 `stats.lessonsByBook`（各册已收录教材课数）。
- 刷题页册数 chips 显示实际题量（如「第2册 20题」），题量少于 50 的册 hover 提示"持续扩充中"；教材学习册 chips 显示收录课数或「未收录」（`.chip-count` 样式）。
- 页脚改为如实描述："当前完整收录第一册（教材 + 题库），第 2–4 册持续扩充中"。

## 2. 「今日学习」首页 + 标签归并

**问题**：10 个标签平铺无引导，新用户不知从哪开始；「生词本」与「背单词」概念重叠。

**改动**：
- 新增默认落地页「🏠 今日学习」（`#homePanel`），四张卡片：今日目标进度条+连续打卡（`/api/plan/overview`）、到期复习数（`/api/srs/stats`）、继续上次学习的课（localStorage `nce-last-lesson`，由 openLesson 记录）、累计概况+快捷入口（听写/背单词/阶段测验/薄弱分析）。
- 标签栏按 **学 / 练 / 复习 / 我的** 分组（`.tab-group`，`TAB_GROUP_OF` 映射）；registerFeature 自动把功能标签挂入对应组，未列出的进「我的」。
- **生词本 + 背单词合并**为一个「🔤 单词」标签（`MERGE` 配置），内部子页切换（`.subtabs`/`.subpanel`），两个功能模块文件零改动。
- 新增 `NCE.gotoTab(id)` 支持跳转任意标签（含合并子页）。

## 3. 答题流程：跨册 bug 修复 + 草稿保存 + 逐题模式

- **Bug 修复**：`retryWrong`（只练本次错题）原来按"当前选中册"重新拉题再过滤，错题本跨册重练时会静默丢题；改为直接从本次已加载的 `state.questions` 过滤。
- **草稿保存**：作答内容实时存 sessionStorage（`nce-quiz-draft`），页面刷新后自动恢复题目与已填答案（整卷/逐题均支持）；提交或点「返回设置」清除。
- **逐题模式**：出题设置新增「作答模式」（整卷提交 / 逐题反馈）。逐题模式一题一判（`/api/grade` 单题提交），即时显示对错+解析（`.step-feedback`），做完出聚合成绩单；错题同样进 SRS 队列。
- **重要修复：选项打乱**。题库源文件 99% 的单选答案排在第一个选项且原样下发，可被「永远选A」破解。新增 `lib/data.js#publicQuestion`（去答案/解析 + Fisher-Yates 打乱选项），`server.js`、`routes/srs.js`、`routes/exam.js` 三处下发路径统一改用。判分按答案文本比对，不受选项顺序影响。

## 4. 交互细节

- **TTS 语速调节**：顶栏新增语速选择（慢速/较慢/常速/快速），localStorage 记忆，`speak`/`speakSequence` 统一生效（含听写、单词朗读），切换时朗读一句试听。
- **逐题模式键盘操作**：数字键 1-4 选选项、回车确认/下一题；答题卡内有键盘提示。
- **toast 轻提示**：新增 `toast(msg, type)` 并暴露为 `NCE.toast`，替换 app.js 中 6 处阻塞式 `alert()`；破坏性操作（重置进度、未答完提交）仍保留 `confirm()`。
- 本轮并行完成（另一会话）：教材目录已学标记、←/→ 翻课、主题切换（themes.css/theme.js）——本轮改动已与其兼容（语速控件避开了 theme.js 的 `insertBefore` 注入点）。

## 验证

- `npm run validate`：0 错误 0 警告；`node --check` 通过。
- 无头 Chrome E2E（puppeteer-core）全部通过：首页四卡渲染、标签分组、单词子页切换、册数题量标注、草稿刷新恢复、整卷提交清草稿、错题重练、逐题即时反馈与成绩单、数字键/回车键盘操作、toast 显示、选项打乱抽样（下发首选项与源文件一致率 25% ≈ 随机水平）且无答案字段泄露。
